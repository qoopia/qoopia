# Фаза 1: Principles

**Цель фазы**: до того как смотреть код и схему, ответить на вопросы "зачем Qoopia существует" и "что для неё значит 'хорошо работает'".

Без этого документа любой дальнейший аудит будет субъективен — мы будем судить систему по чужим меркам, а не по меркам владельца.

## Документы

- `01-why.md` — Зачем Qoopia. Какую боль она убирает. Что было бы, если её не было. ⚪
- `02-personas.md` — Кто пользуется. Роли, ожидания, контекст использования. ⚪
- `03-use-cases.md` — Что они делают, в каком порядке, как часто. ⚪
- `04-success-criteria.md` — Что мы считаем "хорошо работает". Метрики и ощущения. ⚪
- `05-non-goals.md` — Чем Qoopia **не** должна быть. Где заканчивается её зона. ⚪

## Процесс

Документы пишутся через диалог: Claude задаёт вопросы → Асхат отвечает → Claude фиксирует → Асхат корректирует → фиксируется финал → идём к следующему.

**Не пишем "от себя".** Всё что попадает в эти документы должно быть либо прямо сказано Асхатом, либо им явно подтверждено.

## Exit criteria Фазы 1

- [ ] Все 5 документов написаны
- [ ] Асхат явно сказал "да, это про меня" на каждый
- [ ] Подписан ADR-002: "Принципы приняты"
- [ ] Только после этого начинается Фаза 2 (AS-IS)
